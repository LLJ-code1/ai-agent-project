# 俊诚风格:工作汇报 / 机制建设 PPT 模板

> 俊诚风格是一套偏"领导汇报"的页面语言,适合讲项目推进、机制建设、流程优化和资源诉求。它不等同于宏观投研观点页风格,不要直接混用。

---

## 一句话定义

俊诚风格的核心是:

> 把复杂工作讲成"问题明确、路径清楚、支持可要、效果可衡量"。

它不是以装饰感取胜,而是以**问题意识、推进路径和机制表达**取胜。

---

## 适用场景

适合:

- 工作汇报
- 项目规划
- 机制建设
- 流程优化
- 优化前 / 优化后复盘
- 自动化工程建设说明
- 需要向领导说明"为什么做、怎么做、需要谁支持"的页面

不适合:

- 每日市场观点页
- 单页资产判断页
- 纯经济数据解读页
- 需要克制、研报式表达的投研结论页

---

## 标准叙事结构

一页通常按以下顺序组织:

```text
标题: 规划 X / 机制名称 / 工作主轴

1 解决什么问题?
  用一句问题句讲清痛点

2 具体怎么做?
  用流程、机制图、矩阵、优化前后对比讲清路径

3 需要什么支持?
  底部支撑栏写清楚需要哪些团队、数据、系统或权限

可选: 数据表现 / 效果证明
  用阅读量、点击量、覆盖数量、耗时下降、产出频次等证明价值
```

## 视觉语言

### 标题区

- 左上保留红色竖条或红色强调线
- 大标题用黑色粗体
- 关键机制名或核心词用招商红强调
- 右上可放淡灰 Logo 或项目标识

示例:

```text
规划 4: 建立"宏观投研日报自动化机制"
```

### 问题区

- 用深蓝横条承载
- 左侧带序号 `1`
- 问题句要直指痛点
- 关键对象或痛点可用黄色强调

示例:

```text
如何把分散宏观数据转化为稳定、可复用的投研日报?
```

### 方案区

常用组件:

- 流程箭头
- 机制图
- 优化前 / 优化后对比
- 多角色泳道
- 左右双栏
- 中心菱形或枢纽节点
- `01 / 02 / 03` 编号结论

方案区可以高密度,但必须有清晰阅读顺序。

### 支持区

- 页面底部放一条灰蓝色支撑栏
- 左侧写 `3 需要什么支持?`
- 右侧写具体支持方和资源
- 关键支持方用黄色强调

示例:

```text
需要什么支持?  Wind 数据权限、指标口径确认、飞书推送权限、投研反馈机制
```

## 推荐页面类型

### 1. 问题-方案-支持页

用于介绍一个新机制。

结构:

```text
问题栏
方案流程图
底部支持栏
```

适合本项目:

- 宏观日报自动化机制
- PPT/HTML 输出模板化机制
- 飞书推送机制

### 2. 优化前 / 优化后页

用于展示工作价值。

结构:

```text
左: 优化前
中: 三个优化动作
右: 优化后
底部: 效果或支持
```

适合本项目:

- 人工整理数据 → 自动导出 JSON
- 临时写观点 → 固定 Prompt 和数据边界
- 手工排版 → 模板化 PPT/HTML

### 3. 工作机制流程页

用于讲清链路和责任分工。

结构:

```text
触发条件 → 数据来源 → 内容确定 → 内容制作 → 审核/复核 → 发布
```

适合本项目:

```text
Wind 刷新 → Excel 打分 → JSON 导出 → AI 分析 → PPT/HTML 渲染 → 飞书推送
```

### 4. 数据表现页

用于证明机制有效。

可用指标:

- 覆盖指标数量
- 覆盖资产数量
- 生成耗时
- 人工步骤减少数量
- 日报产出频次
- 用户阅读/点击/反馈

## 用在本项目时的典型表达

### 项目介绍页

```text
解决什么问题?
如何把 Wind 宏观数据转化为稳定、可复用、可追溯的投研日报?

具体怎么做?
Wind 刷新 → Excel 自动打分 → JSON 快照 → AI 分析 → PPT/HTML 展示 → 飞书推送

需要什么支持?
Wind 数据权限、指标口径确认、投研观点反馈、飞书接口权限
```

### 优化前后页

```text
优化前:
- 数据分散在 Wind 和 Excel
- 分析依赖人工临时组织
- 页面排版依赖手工编排

优化后:
- 118 个指标进入固定节点体系
- 7 条资产链路自动形成信号流
- JSON / AI / PPT / HTML 逐步模板化
```

## 使用边界

1. 俊诚风格服务"汇报工作",不直接服务"市场观点"。
2. 页面允许信息密度高,但必须保留 `问题-方案-支持` 的阅读顺序。
3. 所有机制描述要落到可执行动作,不要只写抽象口号。
4. 如果页面里出现数据,必须说明数据口径或来源。
5. 如果用于宏观自动化项目,不得越过 Excel/JSON 的数据边界做效果归因。

---

## 和宏观投研风格的区别

| 维度 | 俊诚风格 | 宏观投研风格 |
|------|----------|--------------|
| 主要用途 | 汇报工作、推进项目 | 表达市场观点 |
| 读者关注 | 为什么做、怎么落地、谁支持 | 经济/资产怎么看 |
| 页面结构 | 问题-方案-支持-效果 | 判断-证据-图表-结论 |
| 信息密度 | 可以很高 | 高但要服务观点 |
| 常用图形 | 流程、机制、对比、角色分工 | 图表、热力表、指标卡 |
| 风险 | 过度复杂、像项目说明书 | 堆数据、结论不清 |
