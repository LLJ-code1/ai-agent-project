# 打分系统的设计哲学

## 核心问题:什么是"好的打分"

投研打分系统面临的根本难题是:**同一个指标值对不同资产的含义不同,对同一资产在不同时期的含义也不同**。

比如布伦特原油 90 美元:
- 对 A 股经济:负面(输入性通胀压力)
- 对黄金短期:负面(高油价→加息预期→美元强→金跌)
- 对黄金长期:正面(通胀对冲需求)
- 对俄罗斯股市:正面(出口收入)

一个"僵化的阈值表"必然失效。本项目的打分系统基于 3 条原则:

---

## 原则 1:基于"对资产的含义",不是"指标本身"

**错误做法**:
```
PMI > 50 → +1
PMI > 51 → +2
```
这只反映"经济扩张",不反映"对 A 股的影响强度"。

**正确做法**:
```
PMI > 52 → +2  (显著扩张,强利好)
PMI > 51 → +1  (温和扩张,利好)
PMI > 50 → +0.5 (刚扩张,弱利好)
PMI > 49 → 0   (临界,中性)
PMI > 48 → -1  (温和收缩)
PMI ≤ 48 → -2  (显著收缩)
```

阈值的设计基于**历史上该指标在这些值附近时,A 股同期表现的经验分布**,而不是经济学教科书上的"荣枯线"。

---

## 原则 2:嵌入"预期差"维度

市场对新信息的反应,核心看**预期差**(实际 - 预期),不是绝对值。

如果 PMI 一致预期是 49.5,实际公布 50.5,市场反应 = "好于预期 1 个点",可能引发上涨。
如果预期就是 50.5,实际也是 50.5,市场反应 = "符合预期",可能不涨甚至回调。

**本项目的处理**(对有一致预测值的指标):

```excel
=IF(M12="",
    [绝对值打分],     // 没有预期值时,fallback 到绝对值
    IF(H12-M12 >= 0.5, 2,
       IF(H12-M12 >= 0.2, 1,
          IF(H12-M12 >= -0.2, 0,
             IF(H12-M12 >= -0.5, -1, -2)
          )
       )
    )
)
```

M 列 = 预期值,H 列 = 实际值。

---

## 原则 3:加入"变化方向"加分

在绝对值打分基础上,叠加**变化方向**的 ±0.5 加减分:

```excel
基础分 = [绝对值打分]
方向加分 = IF(L23 < -0.05, +0.5,    // 数据环比下降显著 → 对债市利好 → +0.5
              IF(L23 > 0.05, -0.5,   // 数据环比上升显著 → 对债市利空 → -0.5
                 0))
最终得分 = 基础分 + 方向加分
```

**为什么**:即使绝对值在中性区间(如 PMI 刚过 50),但**方向明确**(连续 3 个月上行)也应该给出信号。

---

## 阈值的"不对称性"

投研实践中发现:**市场对负面信息的反应比正面信息更敏感**。同样幅度的好数据/坏数据,坏数据触发的下跌幅度通常大于好数据的上涨幅度。

因此本项目的部分阈值故意设计为**不对称**:

```
极限正分(+2)需要很强的信号
极限负分(-2)门槛相对低一些
```

比如美国非农:
- ≥ 250 万 → +2(显著强)
- ≥ 150 万 → +1(正常)
- ≥ 100 万 → 0(偏弱)
- < 100 万 → -1(疲软)
- < 0(负值)→ -2(衰退信号)

---

## 节点得分的合成规则

### 加权平均(不是简单平均)

每个节点的得分 = Σ(指标打分 × 权重),权重总和 = 1。

**权重的设计**:
- **重要指标**权重大(如社融 0.35)
- **高频噪音指标**权重小(如 DR001 0.05)
- **方向性一致**的指标合并计算权重(避免信息冗余)

### 议题 1:主项 / 子项原则

同一节点内:
- **主项**(3-5 个):权重 > 0,参与计算
- **子项**(多个):权重 = 0,仅在 JSON 中展示,供 AI 做结构分析参考

**例**:A 股 CN-fin 实体传导节点:

| 指标 | 类型 | 权重 |
|------|-----|------|
| 社融存量同比 | 主项 | 0.35 |
| 新增人民币贷款 | 主项 | 0.20 |
| M1 | 主项 | 0.25 |
| M2 | 主项 | 0.20 |
| 居民短贷 | 子项 | 0 |
| 居民中长贷 | 子项 | 0 |
| 企业中长贷 | 子项 | 0 |
| 票据融资累计 | 子项 | 0 |
| 居民贷款累计 | 子项 | 0 |
| 短贷及票据累计 | 子项 | 0 |
| 未贴现票据 | 子项 | 0 |
| 企业债券融资 | 子项 | 0 |
| 政府债券融资 | 子项 | 0 |
| 非银贷款 | 子项 | 0 |

**为什么这么设计**:
1. 避免"双重计算"(新增贷款本身已经包含居民、企业分项)
2. 让节点得分纯粹反映"总量维度",结构分析交给 AI 的文字描述
3. 参考 Conference Board LEI 的"10 独立成分"原则

---

## 链路总分的合成

```
链路总分 = ∑(节点得分) / 节点数
```

**为什么是简单平均,不是加权**:
- 每个节点本身已经是"综合指标",权重已经在节点内体现
- 不同链路的节点数量不同(4-5 个),简单平均更公平
- 避免"二级权重"的复杂性

**链路总分的解读**:
- **不代表买/卖建议**
- 反映**链路内部一致性和信号强度**
- 正分意味着链路逻辑顺畅支持多头方向
- 负分意味着链路内部存在矛盾或断点
- **真正有信息量的是信号流的节点组合,而非总分数字本身**

---

## 信号标签的设计

每个节点得分映射为 emoji + 文字:

| 得分区间 | emoji | 典型文字(语境化) |
|---------|-------|-------------------|
| ≥ +1.5 | ☀☀ | 显著宽松/大幅流入/显著乐观 |
| +0.5 ~ +1.5 | ☀ / ⛅☀ | 温和宽松/偏好/温和活跃 |
| -0.5 ~ +0.5 | ⛅ | 常规/中性/企稳观望 |
| -1.5 ~ -0.5 | 🌧 / ⛅🌧 | 传导不畅/偏紧/略偏贵 |
| ≤ -1.5 | 🌪 | 显著偏贵/恶化 |

文字不是固定的,不同节点用不同表述:
- 央行节点:宽松 / 常规 / 紧缩
- 银行间节点:极宽松 / 宽裕 / 偏紧
- 市场节点:活跃 / 温和 / 低迷
- 估值节点:便宜 / 合理 / 偏贵

---

## 打分系统和分析产出的关系

**打分是"骨架",AI 分析是"肌肉"**:

- 打分告诉 AI:"这个节点是什么状态" (数字化信号)
- AI 基于打分 + 指标原始数据生成文字分析
- 阈值设计合理 → AI 产出更精准
- 阈值有偏 → AI 产出可能被带偏

所以 **打分阈值的迭代和 AI Prompt 的迭代是协同的**。任一边的改动都要同步考虑另一边的影响。

---

## 阈值的回测与迭代

**当前状态**:打分阈值基于投研员的经验 + 历史数据观察,没有严格的回测。

**未来计划**:
- 用历史 JSON 快照做回测(把每日链路得分 vs 次日资产涨跌对比)
- 如果某节点得分的预测力弱,说明阈值需要调整
- 迭代阈值 → 重新回测 → 迭代阈值 → ...

**注意**:投研打分不追求"精确预测",而是"提供结构化的决策辅助"。即使回测 AUC = 0.55(略好于随机),也能为分析师节省大量机械工作时间。

---

## 最后的原则

> 打分系统是工具,不是神谕。
> 分数的变化趋势比数字本身更有意义。
> AI 的任务是"把打分翻译成专业判断",不是"被打分绑架"。

有时分析师看到 +0.5 的得分,基于其他信息(如地缘事件、政策窗口)判断应该更重视或更轻视这个信号,这是**应当鼓励**的人工覆写。打分只是输入之一,不是最终答案。
