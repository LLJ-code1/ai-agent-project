# 项目架构

## 四层金字塔

整个分析框架按"自下而上"的信息聚合逻辑设计:

```
                    ┌──────────────┐
                    │  L0 策略层    │  7 个资产决策
                    │   7 assets    │
                    └───────┬──────┘
                            │
                    ┌───────┴──────┐
                    │ L1 基本面层   │  中国 / 美国
                    │  2 regions    │
                    └───────┬──────┘
                            │
                    ┌───────┴──────┐
                    │ L2 传导层     │  29 个节点
                    │  29 nodes     │
                    └───────┬──────┘
                            │
                    ┌───────┴──────┐
                    │ L3 指标层     │  118 个指标
                    │  118 inputs   │
                    └──────────────┘
```

每层的含义:

- **L3 指标层**:原始 Wind 数据,每个指标打分 0~±2
- **L2 传导层**:把相关指标加权合成一个节点分数,如"央行态度" = 逆回购利率 × 权重 + 存准率 × 权重 + 结构性工具 × 权重
- **L1 基本面层**:把节点串成链路,如 A 股基本面 = 央行 → 银行间 → 实体 → 经济 → 市场
- **L0 策略层**:每个资产的综合判断 = 链路内所有节点得分加总 / 节点数

## 节点的信号标签

每个节点得分对应一个 emoji + 文字的信号标签,比如:

| 得分区间 | emoji | 文字 |
|---------|-------|------|
| ≥ +1.5 | ☀☀ | 显著正面 / 大幅流入 / 极宽松 |
| +0.5 ~ +1.5 | ☀ / ⛅☀ | 温和正面 / 偏好 |
| -0.5 ~ +0.5 | ⛅ | 中性 / 常规 |
| -1.5 ~ -0.5 | 🌧 / ⛅🌧 | 温和负面 / 偏弱 |
| ≤ -1.5 | 🌪 | 显著负面 / 恶化 |

具体信号文字会根据节点语境不同(比如"央行"用"宽松/紧缩","市场"用"活跃/低迷")。

## 信号流的"断点"

链路中如果出现**前后信号强烈不一致**,就是所谓的"断点":

**例 1:A 股的银行间 vs 实体断点**
```
银行间☀☀ 显著宽松  →  实体⛅🌧 传导不畅
```
含义:钱在银行间堆积,但没流向实体,可能是"流动性陷阱"特征。

**例 2:美股的基本面 vs 估值断点**
```
美国经济⛅ 企稳  +  估值🌪 显著偏贵
```
含义:基本面支持股价,但估值已偏贵,风险收益比不对称。

**断点是分析的起点**,AI 产出分析时会优先展开断点讨论,而不是按节点顺序流水账。

## 议题 1 权重重构后的"主项/子项"原则

同一节点内的指标分两类:

- **主项**(保留权重):能独立反映一个维度的指标,如社融、新增贷款
- **子项**(权重=0):已有主项的分解或同义指标,如居民中长贷(是新增贷款的分解)、总 CPI(是核心 CPI 的上位概念)

**为什么这么做**:
- 参考 Conference Board LEI 的"去重"原则——10 个成分都是独立维度
- 避免父子项同时加权造成"双重计算"
- 让节点得分更纯粹反映"不同维度的综合信号",而不是"同一维度的多次计算"

## 打分系统的核心原则

完整设计哲学见 `04_scoring_logic.md`。这里只列 3 条:

1. **打分基于"对资产的含义"而非"指标本身好不好"**
2. **嵌入"预期差"维度**(有一致预测时)
3. **加入"变化方向"加分**(在绝对值基础上)
