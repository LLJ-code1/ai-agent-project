# 项目全貌

## 这个项目解决什么问题

投研团队的日常工作流程中,每天早上要做的事:

1. 打开 Wind 终端,拉 100+ 个宏观指标
2. 人工整理,对比预期、历史
3. 分链路判断对不同资产的影响
4. 写成研报,发给同事或发布

**这个过程每天要花 2-4 小时**,主要耗在"机械整理"和"排版",真正的专业判断只占一小部分时间。

本项目的目标是:**把机械部分自动化,把分析师的时间留给真正需要判断的部分**。

## 使用者画像

**主要使用者**:投研团队一名分析师

- **非开发者**:不写代码,不懂 Excel 公式的细节
- **专业投研背景**:能看懂所有金融术语、研报逻辑
- **Wind 终端熟练**:每天手动刷新数据
- **决策者**:项目要不要改、怎么改、加什么指标,由他说了算

**间接使用者**:团队其他成员 + 领导。他们只看最终产出(HTML / 飞书推送),不接触中间环节。

## 核心工作流

```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│ ① Wind 刷新 │ → │ ② Excel 打分 │ → │ ③ 导出 JSON │
│   (人工)    │   │   (全自动)   │   │   (全自动)   │
└──────────────┘   └──────────────┘   └──────────────┘
                                              │
                                              ▼
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│ ⑥ 飞书推送  │ ← │ ⑤ HTML 展示 │ ← │ ④ AI 分析   │
│  (未开发)   │   │  (半自动)   │   │  (全自动)   │
└──────────────┘   └──────────────┘   └──────────────┘
```

**当前状态**:5/6 环节已自动化,从 Wind 刷新到分析产出约 5 分钟,相比过去约 2-4 小时。

## 项目的本质:"结构化转换"

这套系统的技术本质是**数据结构化转换**,底层是一个**四层金字塔**(详见 `docs/02_architecture.md`):

```
原始 Wind 数据(海量数值)
  ↓ 打分函数
打分信号(0~±2,每指标一个)
  ↓ 加权公式
节点得分(0~±2,每节点一个)
  ↓ 串联组合
链路信号流(一句话描述,每链路一条)
  ↓ AI 生成
结构化分析(markdown 报告,每链路 600-800 字)
  ↓ HTML 渲染
可视化展示(单文件 HTML)
```

每一步都是**信息压缩 + 语义添加**:从冰冷数字到带判断的文字。

## 产出物

### 每次跑的产出

- **JSON 快照**(完整版 + 精简版)
- **7 份 markdown 分析**
- **1 份 HTML 展示页**

### 沉淀的产出

- **标准化的分析框架**(Prompt 规则 + 打分阈值)
- **可追溯的指标字典**
- **可迭代的知识库**(本仓库所有文档)
