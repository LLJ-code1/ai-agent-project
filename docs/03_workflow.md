# 6 环节工作流详解

## 概览

```
① Wind 刷新 → ② Excel 打分 → ③ 导出 JSON → ④ AI 分析 → ⑤ HTML 展示 → ⑥ 飞书推送
   (人工)     (全自动)      (全自动)       (全自动)     (半自动)     (未开发)
```

每次跑完一轮约 5 分钟,相比过去人工整理节省 2-4 小时。

---

## 环节 ①:Wind 数据刷新(人工触发)

**做什么**:投研员打开 `macro_final_v11.xlsx`,在 Wind 插件里点"刷新"。

**背后发生什么**:Wind 把 118 个指标的最新值写入"宏观数据"、"指数走势"、"A股成交额换手率测试1" 这 3 个 Sheet。

**产出**:刷新后的 Excel 文件。

**用时**:约 1 分钟。

**为什么是人工**:Wind 终端本身不开放 API,只能手动触发。这是目前唯一的人工环节。

---

## 环节 ②:Excel 自动打分(全自动)

**做什么**:刷新 Wind 后,在 Excel/WPS 里**保存一次**,触发所有公式重新计算。

**背后发生什么**:

1. 汇总 Sheet 的 H 列(当前值)用 `LOOKUP(9.99E+307, ...)` 自动从宏观数据 Sheet 取最新值
2. O 列(打分)用嵌套 IF 根据阈值计算 0~±2 分
3. 传导链路 Sheet 的 E 列(节点得分)用加权公式 = Σ(权重 × O列) 计算
4. F 列(信号标签)根据节点得分映射为 emoji + 文字
5. 总览行的 F 列拼接 5 个节点的信号,生成信号流

**产出**:
- 汇总 Sheet 有 118 行打分 + 信号
- 传导链路 Sheet 有 29 个节点得分 + 7 条链路信号流

**用时**:约 10 秒(取决于机器性能)。

**为什么用 Excel**:
- 投研员本就熟练 Excel,改权重、改阈值可以自己改不用等开发
- 公式是透明的,可以逐单元格追溯问题
- 不依赖服务器、不依赖网络

---

## 环节 ③:Python 导出 JSON(全自动)

**做什么**:运行 `python macro_snapshot_export.py macro_final_v11.xlsx`。

**背后发生什么**:

1. **前置检查**:抽查 R25/R60/R75/R105 几个单元格,确认 Excel 已被 recalc(openpyxl 只能读缓存公式值)
2. **读取汇总 Sheet**:把 118 行数据提取为 dict
3. **按 NODE_INDICATORS 映射组装 29 个节点**:每个节点指向 n 个指标的 dict
4. **计算衍生指标**:如 10Y-2Y 利差、金油比等从其他指标实时计算
5. **读取 history 字段**:从宏观数据/指数走势 Sheet 取最近 12 个月(月度)或 60 天(日度)的历史值
6. **输出两份 JSON**:
   - 完整版(约 500KB,含全部 history)
   - 精简版(约 150KB,约 23k tokens,供 AI 使用)

**产出**:
- `snapshot_YYYY-MM-DD.json`(完整版)
- `snapshot_YYYY-MM-DD_lite.json`(AI 精简版)

**用时**:约 5 秒。

**为什么分两版**:
- 精简版专供 AI,tokens 可控
- 完整版给 HTML 展示(需要 history 画趋势图)或给人类直接看

---

## 环节 ④:AI 生成分析(全自动)

**做什么**:把精简版 JSON + `analysis_prompt_v2.2.md` 投喂给大模型(Claude/GPT/Gemini 等),让其产出 7 份分析。

**背后发生什么**:

1. **Prompt 告诉 AI 写作框架**:
   - 信号流(带节点名)开头
   - 全景判断(不超 100 字)
   - 断点/堵点展开
   - 节点补充
   - 后续关注

2. **Prompt 告诉 AI 硬规则**:
   - 数据边界、日度数据表述、语气规范、术语通俗化、价格指标溯因、数值比较
   - 禁用词:极度、暴涨、崩塌、恶化、一定、必然等

3. **AI 可以调用搜索工具补充**:
   - Bloomberg 港股盈利预期(Wind 覆盖不全)
   - CME FedWatch 降息概率(Wind 没这个数据)
   - 地缘新闻、美联储讲话

4. **产出 7 份 markdown**,每份 600-800 字,覆盖 A 股 / 港股 / 短债 / 中长债 / 美股 / 美债 / 黄金。

**产出**:7 份 `分析_<资产>_YYYY-MM-DD.md`

**用时**:视模型和搜索复杂度,约 2-5 分钟。

**质量校验规则**:详见 `skills/analysis_writing.md`。

---

## 环节 ⑤:HTML 展示(半自动)

**做什么**:把 7 份分析 + JSON 的关键数据 + 21 个趋势图组装成一个单文件 HTML。

**背后发生什么**(当前是 AI 手工编排):

1. **生成 sparkline 趋势图**:
   - 月度数据 12 点折线
   - 日度数据 60 点折线
   - 双轴图(量纲不同的两条线)
2. **嵌入到分析 body 的对应位置**:每个图紧贴相关论点
3. **组装完整 HTML**:Hero + Pipeline 流程图 + 方法论 + 仪表盘 + 7 链路分析(A股/美股默认展开)+ 跨链路洞察 + 路线图
4. **保存为单文件** `项目展示_YYYY-MM-DD.html`,可离线分发

**产出**:`项目展示_YYYY-MM-DD.html`(约 100KB)

**用时**:当前由 AI 手工做,约 10-20 分钟。议题 2 完成后会抽象为 Python 脚本,直接模板化生成。

**视觉风格**:暗色投研终端风(#0a0e14 底色 + #f0b90b 琥珀强调色),类彭博/Refinitiv。

## 环节 ⑥:飞书推送(未开发)

**目标**:每日定时把 HTML 或摘要推送到投研团队飞书群。

**预期功能**:
- 定时任务:每个交易日 9:30(开盘前)自动跑完整流水线
- 飞书消息:图片摘要 + 详细 HTML 链接
- 异常处理:Wind 数据异常、公式报错时通知开发者

**当前优先级**:P3,需要先做完议题 2(AI 打分)再做。

## 各环节的责任边界

| 环节 | 谁负责 | 谁修改 |
|------|-------|-------|
| ① Wind 刷新 | 投研员 | 投研员 |
| ② Excel 打分 | Excel 公式 | AI(加指标时)/ 投研员(改阈值时) |
| ③ Python 导出 | 脚本 | AI(行号变动时) |
| ④ AI 分析 | AI + Prompt | AI(改 Prompt 时) |
| ⑤ HTML 展示 | AI | AI |
| ⑥ 飞书推送 | 未开发 | 未开发 |

## 哪些环节已经"稳定",哪些还在"迭代"

**稳定**(已跑通 N 轮,出问题概率低):
- ① Wind 刷新
- ② Excel 打分(v11 + 议题 1 重构后)
- ③ Python 导出(v2.2)

**迭代中**(频繁调整):
- ④ AI 分析(Prompt 还在细调,如刚加的"数值比较硬规则")
- ⑤ HTML 展示(还没完全模板化,每次有新需求要 AI 重新编排)

**未开发**:
- ⑥ 飞书推送
