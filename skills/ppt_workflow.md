# PPT 制作操作手册

> 触发词：PPT、slide、deck、汇报页、展示页、路演稿、投研汇报。

---

## 使用场景

当用户要求把 Excel、JSON 或分析观点做成 PPT 时，先读：

1. `AGENTS.md` 或 `CLAUDE.md`
2. `AGENT_PROTOCOL.md`
3. 本文件
4. `docs/06_ppt_workflow.md`
5. `docs/07_presentation_system.md`

PPT 任务仍遵守项目协议：先讲方案，等用户确认，再动手。

## 工作流程

### 1. 明确页面目标

先问清或自行归纳：

- 这页讲哪条传导链路？
- 这页回答什么问题？
- 是总览页、证据页、价格页，还是流动性页？
- 是投研观点页，还是工作汇报/机制建设页？

不要把多个主题塞到同一页。例如 A 股经济基本面页不应混入社融、M1、M2。

### 1.1 选择输出风格

做 PPT 前必须先判断使用哪套风格：

| 页面类型 | 使用风格 | 参考模板 |
|----------|----------|----------|
| 日报、研报、经济/资产观点 | 宏观投研风格 | `templates/ppt/a_share_economy_basic.md` |
| 宏观框架页、资产映射页 | 宏观投研风格 | `templates/ppt/macro_asset_framework_page.md` |
| 工作汇报、项目推进、机制建设 | 俊诚风格 | `templates/ppt/juncheng_work_report_style.md` |

判断规则：

- 如果页面回答"经济怎么看 / 资产怎么看",用宏观投研风格。
- 如果页面回答"项目解决什么问题 / 机制怎么搭 / 需要什么支持",用俊诚风格。
- 同一个 deck 可以分章节切换风格,但单页内不要混用两套设计语言。

### 2. 校准数据

动手前必须回到 Excel/JSON 核对最新值：

- 当前值
- 上期值
- 数据日期
- 单位
- 是否有可用历史序列

尤其注意价格指标。PPI 转正、PMI 原材料价格上行、油价变化不能凭印象判断。

### 3. 先写页面判断

每页先写 1 个主判断和 2-3 条支撑判断，再选图表。

示例：

```text
主判断：PPI 转正偏输入，内需修复仍需观察。
支撑 1：PPI 由 -0.9% 转为 0.5%。
支撑 2：PMI 原材料购进价格由 54.8 升至 63.9。
支撑 3：社零 1.7%、商品房销售面积 -10.4%、民间固定投资 -2.2%。
```

### 4. 选择图表

| 问题 | 推荐图表 | 注意事项 |
|------|---------|---------|
| 多指标全景强弱 | 热力表 | 先分组，再看颜色 |
| PMI 分项差异 | 折线图 | 纵轴压窄，标 50 荣枯线 |
| PPI/CPI 变化 | 折线图 | 可同轴，但要控制范围 |
| 原材料价格 | 单独折线或副轴 | 不与 PPI/CPI 硬放同轴 |
| 内需约束 | 上期→本期对比表 | 不要被单个大负值压缩视觉 |

### 5. 制作并渲染检查

交付前必须检查：

- 标题是否只出现一次
- 图表是否能看出结论
- 文字是否挤压或重叠
- 坐标轴是否误导
- 表格是否有水印或无关文字
- 数据是否与 Excel/JSON 一致
- 标题或说明是否被任何框体遮挡
- 文本是否出框
- 上下连接线/箭头是否同一种语言
- 左右版心是否平衡
- 是否残留上一版旧文案
- 指标是否放对框

## 设计硬规则

### R-PPT.1 一页一个主判断

每页只讲一个主判断。不要为了信息完整，把经济、流动性、市场交易、估值放在同一页。

### R-PPT.2 图表必须回答问题

每个图表都要能回答一句话。如果读者看不出图在说明什么，改成表格或删除。

### R-PPT.3 坐标轴要主动设计

默认坐标轴经常不适合投研图表。PMI 这类窄区间指标，不能从 0 开始画，否则线会挤成一团。

### R-PPT.4 不同量级不要硬放同轴

PPI/CPI 是同比百分比，PMI 原材料价格是景气指数。除非使用副轴，否则不要硬放在一张同轴折线图里。

### R-PPT.5 热力表必须分组

热力表指标多时，必须增加分类或分组：

- 领先指标
- 同步指标
- 滞后指标
- 流动性指标

具体分组要服务页面主题。

## A 股经济基本面页模板

### 第一页：总览

结构：

```text
标题：基本面：数据边际企稳，但底层需求仍需观察
三条判断：
1. 从数据看，经济边际企稳
2. 但底层需求仍需观察
3. 结构压力仍在
下方：分类热力表
```

热力表分组：

| 分类 | 指标 |
|------|------|
| 领先指标 | 制造业 PMI、非制造业 PMI、PMI 原材料购进价格 |
| 同步指标 | 工业增加值、固定资产投资、制造业投资、基建投资、地产投资、商品房销售面积、社零、出口 |
| 滞后指标 | CPI、PPI |

### 第二页：证据拆解

结构：

```text
标题：PPI 转正偏输入，内需修复仍需观察
三条判断：
1. 价格转正不等于需求确认
2. PMI 修复偏生产端
3. 内需基础仍待夯实
三块图表：
左：PMI 分项折线图
中：PPI/CPI + 原材料价格
右：内需指标 2月→3月对比表
底部：核心判断框
```

## 宏观框架页模板

适用场景：

- 国内宏观至资产
- 美国宏观至资产
- 流动性到多资产映射

固定结构：

```text
左：流动性大框 + 经济大框
右：资产观察 + 独立策略框
```

固定规则：

- 左侧讲传导,右侧讲映射
- 右侧每个资产一行,固定为“观察框 → 策略框”
- 策略框建议拆成“结论 + 操作”
- 不要把策略写进观察框
- 同一页上下连接线/箭头语言保持一致

常见错误：

- 国内页套到美国页后,旧文案没删干净
- 只改标题,不改指标归类
- 上面细连接,下面粗箭头
- 左边主框太窄,版心失衡

## 常见问题

### 问题 1：用旧印象写数据

错误：没有核对 Excel，就写“PPI 仍为负”。

正确：先核对最新值。若 PPI 已转正，再分析转正来源。

### 问题 2：PMI 折线图看成一团

原因：坐标轴范围过宽。

正确：设置 44-52 这类窄区间，并标出 50 荣枯线。

### 问题 3：横条图看不出内需约束

原因：地产销售等大负值压缩了其他指标。

正确：改成上期到本期的对比表，同时写出“当前仍弱”和“边际变化”。

### 问题 4：混入不属于本页的指标

例子：A 股经济基本面页加入社融、M1、M2。

正确：这些指标放到信用/流动性页，不放到经济基本面总览页。

## 交付要求

PPT 生成物默认放在本地 `outputs/`，不直接入库。只有用户明确要求保存样例 PPT 时，才考虑放入 `samples/`；若用户明确确认“这一版是稳定案例”，可放入 `samples/ppt/`。

项目文件被修改后，仍需按协议更新：

- 对应文档
- 本 skill
- `changelog/YYYY-MM-DD_主题.md`
- git commit + push
