## 议题 1 权重重构要点(2026-04-17)

### 重构原则

参考 Conference Board LEI、高盛 FCI、中金 信贷脉冲 等专业方法论,确立**"主项留权重、子项权重=0"**的操作原则:

- 同一节点内,只给"主项"(如社融、新增贷款、M1、M2)分配权重
- "子项"(如信贷结构 4 分项、PMI 购进价格 等)权重=0 但保留展示
- 消除了原先 4 处父子项重复加权

### 汇总 Sheet N 列全部清零

- 119 行中有值的 113 行 N 列**全部设为 0**
- P 列公式改为 `=IF(O="",0,N*O)`,全部=0
- N 列**不再参与任何计算**,只作标记
- 加权逻辑全部集中到**传导链路 Sheet 的 E 列**(唯一计算源)

### 重构涉及的 5 个节点

| 节点 | 位置 | 变化 |
|------|------|------|
| CN-fin 实体传导 | 传导链路 R6 | 11 指标→4 主项(社融 0.35/新增贷款 0.20/M1 0.25/M2 0.20) |
| CN-eco 经济效果 | 传导链路 R7 | 13→11,加入 R4 非制造业 PMI、R6 固投;删除 R3/R7/R8/R9 四个子项 |
| US-eco 美国经济 | 传导链路 R29 | 11→13,加入 R91/R92(地产)、R100(通胀预期);删除 R98 总 CPI |
| US-liq 美国流动性 | 传导链路 R30 | 4→3,删除 R104 SOFR |
| UB-fed 美联储政策 | 传导链路 R35 | 2→2,删除 R104 SOFR,加入 R116 联储资产(YoY) |

### R116 联储资产整行改造

- 原:绝对值(67 万亿美元),无打分公式
- 新:52 周同比增速(复用 R105 银行贷款的 LOOKUP 模板),打分公式按 ±5%/±1% 阈值
- 频率:月度→周度(修正错误)
- Q/R 列 TEXT 格式用 `"+0.0;-0.0"` 处理负数叠加 bug

### E 列(节点得分)公式结构

每个节点的 E 列公式 = 加权引用汇总 Sheet 的 O 列(原始分)。

**重要:引用的是 O 列(原始分),不是 P 列(议题 1 后=0)。**

示例(议题 1 后的 R6 A 股实体传导):
```
=0.35*汇总!O28+0.20*汇总!O29+0.25*汇总!O38+0.20*汇总!O39
```

权重总和必须=1.0。改权重后用计算器验证。

### F 列(信号流)带节点名

v11 起信号流格式为:`央行⛅️常规操作 → 银行间☀️☀️显著宽松 → ...`

总览行 F 列公式示例(R9):
```
="央行"&F4&" → 银行间"&F5&" → 实体"&F6&" → 经济"&F7&" → 市场"&F8
```

## 公式写法规范

### 取最新值(H 列)

**标准模板**(从宏观数据 Sheet 取):
```excel
=IFERROR(LOOKUP(9.99E+307,宏观数据!$F$XX:$RQ$XX),"")
```

**从指数走势 Sheet 取**(盈利预期):
```excel
=IFERROR(LOOKUP(9.99E+307,指数走势!$G$YY:$IV$YY),"")
```

注意从 `$G$` 开始(跳过 F 列的 wsd 公式)。

### 打分公式(O 列)

**必须用嵌套 IF,不用 IFS**(LibreOffice 不兼容 `_xlfn.IFS`)。

模板:
```excel
=IF(H60="",0,IF(H60<60,1,IF(H60<85,0,IF(H60<100,-0.5,IF(H60<120,-1.5,-2)))))
```

### 52 周同比增速公式(R88 银行贷款 / R116 联储资产)

```excel
=IFERROR((LOOKUP(9.99E+307,宏观数据!$F$XX:$RQ$XX)/INDEX(宏观数据!$F$XX:$RQ$XX,1+(COUNT(宏观数据!$F$XX:$RQ$XX)-53)*7)-1)*100,"")
```

原理:周度数据严格每 7 列一个,COUNT 得到总数据点数,减 53 得到 52 周前的数据点序号。

## 踩坑记录

### 坑 1:IFS 函数不兼容

**现象**:LibreOffice recalc 后大量 `#NAME?` 错误。
**原因**:openpyxl 写入的 `IFS()` 缺少 `_xlfn.` 前缀。
**解决**:所有打分公式改用嵌套 `IF`。

### 坑 2:指数走势 Sheet F 列的 wsd 函数

**现象**:COMEX 黄金 在 LibreOffice recalc 后值为 None。
**原因**:指数走势 Sheet 的 F 列存了 Wind 的 `=wsd(...)`,LibreOffice 不认识。
**解决**:所有从指数走势 Sheet 取值的 LOOKUP,范围从 `$G$` 开始。

### 坑 3:合并单元格阻止写入

**现象**:openpyxl 写入无报错,但保存后单元格仍为空。
**解决**:修改前先 `ws.unmerge_cells()` 解除。

### 坑 4:行号位移导致引用断裂

**现象**:删除/插入汇总 Sheet 行后,传导链路 E 列公式没更新。
**原因**:传导链路 E 列是**手动写的加权公式**,不是 INDIRECT 引用。
**解决**:**永远不要在中间插入/删除行**。新指标追加到末尾。

### 坑 5:双重加权

**现象**:节点得分异常。
**原因**:传导链路 E 列引用了 P 列(加权分)而不是 O 列(原始分)。
**解决**:E 列**只引用 O 列**。议题 1 后 P 列全=0,用 P 也会全=0,但语义错误。

### 坑 6:COMEX 黄金 vs 伦敦现货金

**现象**:金油比 50+,与券商报告 32 差距大。
**原因**:COMEX 期货约 4800,伦敦现货约 2300,差 2 倍。
**解决**:阈值要匹配数据源。当前 COMEX 期货,阈值 ≥35 避险主导。

### 坑 7:Python 脚本必须先 recalc Excel

**现象**:Python 脚本跑出的 JSON 数据没刷新。
**原因**:openpyxl 只能读缓存的公式值。用户 Wind 刷新后如果没**打开 Excel 保存一次**,所有公式还是旧值。
**解决**:Python 脚本开头有前置检查,抽查几个关键单元格(R25/R60/R75/R105),若为 None 报警。

## 修改 Excel 的标准流程

### 新增指标

1. 确认宏观数据 Sheet 中该指标的**行号**
2. 在汇总 Sheet**末尾追加新行**(不要在中间插入)
3. N 列填 0(议题 1 后不再使用)
4. O 列打分公式(嵌套 IF)
5. P 列 `=IF(O{row}="",0,N{row}*O{row})`
6. 在传导链路 Sheet 对应节点的 E 列公式中**加入新行的引用**,重新分配权重
7. **验证节点权重总和=1.0**
8. 用 LibreOffice recalc 验证无错误
9. **同步更新 Python 脚本的 NODE_INDICATORS**
10. 更新 `skills/indicator_management.md` 的指标字典
11. 写变更日志 `changelog/YYYY-MM-DD_add_indicator.md`

### 修改打分阈值

1. 只改汇总 Sheet O 列的对应行
2. LibreOffice recalc 后检查节点得分是否合理
3. 在变更日志里说明"为什么改"

### 修改节点权重

1. 改传导链路 Sheet 对应节点的 E 列公式
2. **权重之和必须=1.0** — 改完后用计算器验证
3. 不要改汇总 Sheet N 列(已全=0)

## 布伦特原油的特殊打分逻辑

布伦特原油(R75)挂在 4 个节点,**所有节点方向一致(油价↑=负分)**:

| 节点 | 逻辑 |
|------|------|
| CN-eco 经济效果 | 高油价→输入性通胀→挤压利润 |
| HK-ext 海外环境 | 高油价→全球风险偏好下降 |
| US-eco 美国经济 | 高油价→通胀上行压制经济 |
| AU-infl 通胀预期 | 高油价→加息预期→美元走强→**金价承压** |

**黄金的 AU-infl 节点,油价也打负分。** 短期逻辑(高油价→加息预期→美元强→金跌)主导,不按长期"通胀利好黄金"打正分。

打分阈值:

| 布伦特价格 | 得分 |
|-----------|------|
| <60 | +1 |
| 60-85 | 0 |
| 85-100 | -0.5 |
| 100-120 | -1.5 |
| >120 | -2 |
