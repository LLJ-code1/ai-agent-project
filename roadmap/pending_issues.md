## 未完成事项(按优先级)

### P1 · 议题 2:AI 替代公式打分

**问题**:Excel 的 IF 阈值打分产出的是"数据陈述"(如"PPI 为 0.5%"),AI 基于这些数据写的分析也是描述性的,缺少"因果与判断"。

**方案**:在 Python 脚本里对关键指标(PPI/CPI/PMI/信贷等有解释空间的)调用 Anthropic API,让 AI 根据上下文生成因果判断(如"PPI 转正或部分受油价推升,成色待察"),替换 JSON 的 `auto_view` 字段。利率、汇率等纯价格指标仍用公式。

**风险**:
- 每次导出 JSON 多一笔 API 成本,要设置 fallback
- AI 产出可能偶尔偏离规则,要加 prompt 约束

### P2 · HTML 研报模板抽象

**问题**:当前 HTML 是 AI 手工编排的,下一轮需要同样的编排要重做。

**方案**:抽象为 Python 脚本直接生成 HTML 模板,从 JSON + 分析 markdown 自动组装。AI 仅负责生成 markdown 内容,不负责排版。

### P3 · 飞书推送集成(第 6 环节)

每日定时,将生成的 HTML 或摘要推送至投研团队飞书群。打通"Wind 刷新 → 团队看到报告"的最后一公里。

### P4 · 知识库文档的可读性优化

**现状**:`skills/excel_maintenance.md` / `skills/indicator_management.md` 里的行号表和权重表密集,长期维护时易出错。

**方案**:
- 把"行号-指标-节点-权重"关系抽出成结构化数据(JSON),文档里只引用不硬编码
- 行号表应以 Python 脚本的 NODE_INDICATORS 为唯一真相来源
